# Cadence Professional

Cadence Professional is the organizational, multi-user professional specialization of the shared Cadence foundation for coordinated project execution.

It is not a generic task tracker with an AI chat box added on top, and it is not the sole owner of Cadence project-management theory. It extends shared project, task, dependency, scheduling, scenario, constraint, projection, conversational, and lens semantics with professional staffing, organizational authority, permissions, approvals, cost, overtime, policy, portfolio consequences, and governance.

The canonical shared doctrine lives in `cadence-strategy`. This repository inherits pressure around:

- projects, tasks, dependencies, sequencing, schedules, priorities, and constraints;
- baseline versus proposed scenarios;
- gap visibility and grounded explanation;
- conversational manipulation through voice or text;
- multiple lenses over the same underlying truth;
- whole-human project pathways and lightweight shared participation;
- visibility, contribution, authority, ownership, and obligation as separable dimensions;
- temporary and malleable groups;
- emergent roles from actual work evidence;
- portable personal experience without employer-secret leakage.

This repository specializes those concepts for professional use rather than redefining them.

## Lightweight professional mode

Not every professional project needs to feel like a formal department or heavily governed program.

```text
TEMPORARY_GROUP != PERMANENT_TEAM
PROJECT_PARTICIPANT != STAFFING_OBJECT
PROFESSIONAL_MULTI_USER != MAXIMUM_ENTERPRISE_CEREMONY
```

A temporary cross-functional group, nonprofit committee, or small organization may need shared purpose, participants, selected visibility, tasks, dependencies, artifacts, decisions, and closeout without immediately activating staffing optimization, cost models, or formal approval chains. Stronger governance should appear only when real consequence justifies it.

## Emergent role recognition and worker continuity

People frequently work outside the boxes defined before work begins.

```text
ASSIGNED_ROLE
from
ACTUAL_TASK_AND_CONTRIBUTION_PATTERN
from
EMERGENT_ROLE_DESCRIPTION
```

Professional Cadence may eventually help surface hidden bridge work, role drift, repeated out-of-role contribution, adjacent capability, and portable experience evidence. These outputs must remain explainable, reviewable, contestable, and separate from automatic employment, compensation, or authority decisions.

## Personal / professional trust-domain boundary

Current inherited implications include:

- personal and professional Cadence may coexist on the same device;
- data, artifacts, memory, permissions, ownership, and AI context must not bleed across trust domains by default;
- same-device availability does not imply cross-domain permission;
- promotion into multi-user Professional and reverse downgrade are governed transitions;
- mixed-context captures may contain both personal and professional meaning;
- offboarding must distinguish the continuing person from organizational access and organization-owned records;
- appropriate portable experience may continue with the person without exporting protected source.

No claim that these behaviors are implemented is made here.

## Foundational product posture

- **Simulation before mutation.** Proposed changes must not silently alter operational truth.
- **Human authorization before consequential execution.** Recommendation, preference, review, approval, authorization, and execution remain distinct.
- **Constraint-aware planning.** Eligibility, dependencies, PTO, deadlines, overtime, costs, policy, and competing priorities are first-class professional inputs.
- **Progressive governance.** Use the smallest sufficient control surface and add stronger machinery only when consequence requires it.
- **Recognition without exploitation.** Worker evidence should surface hidden labor and overload rather than justify unsupported extraction.
- **Organizational throughput without hidden harm.** Optimizing one project must expose displacement, constraint violations, and burnout or fairness pressure.
- **Artifact-centered operational truth.** Projects, scenarios, decisions, approvals, and applied changes should remain durable and attributable rather than ephemeral chat state.

## Current executable proof

Slice 0 is an accepted and merged headless Python proof of:

```text
Observe -> Simulate -> Explain -> Negotiate Constraints
        -> Hold -> Authorize -> Revalidate -> Apply
```

It uses synthetic data and prototype-only semantics. It demonstrates explicit separation among baseline operational truth, simulated scenario state, authorization, revalidation, and applied operational revision.

Run it from `prototype/slice0/`:

```bash
python -m pip install -e ".[dev]"
python -m pytest
python -m cadence_professional_slice0 --fixtures fixtures --output evidence
```

The prototype does not establish a canonical Cadence schema or production scheduling, identity, permission, labor, privacy, or execution semantics.

## Current proposal gate

```text

  -> Revalidate
  -> Apply
```
