# Cadence Professional

Cadence Professional is the multi-user organizational specialization of the shared Cadence foundation.

It owns professional product scenarios, organizational coordination pressure, staffing and task eligibility, permissions and authority, approvals, cost and overtime, portfolio consequences, temporary professional groups, progressive governance, professional trust-domain pressure, and professional-specific implementation evidence.

It is not a generic task tracker with an AI chat box, and it is not the sole owner of Cadence project-management semantics.

## Current implementation

Slice 0 is a headless Python proof of:

```text
Observe -> Simulate -> Explain -> Negotiate Constraints
        -> Hold -> Authorize -> Revalidate -> Apply
```

It uses synthetic data and prototype-only semantics. It demonstrates explicit separation among baseline operational truth, simulated scenario state, authorization, revalidation, and applied operational revision.

Run it from `prototype/slice0/`:

```bash
python -m pip install -e ".[dev]"
python -m pytest
python -m cadence_professional_slice0 --fixtures fixtures --output evidence
```

## Central invariants

- Operational truth is not derived reasoning state.
- A simulated scenario is not live operational truth.
- A recommendation is not authorization.
- Authorization applies to one exact scenario version and bounded change set.
- Applied state remains traceable to baseline, proposal, review, and authorization.
- People are not interchangeable capacity.
- Hidden inference does not become employment or organizational authority.
- Optimizing one project must expose displacement to other work.
- Low-ceremony professional projects should not be forced into maximum enterprise governance.
